# API Versioning Rules

## Principles
- Use semantic versioning for API changes.
- Do not break existing consumers without a deprecation window.
- Document breaking changes in release notes.

## Versioning Strategy
- **Major**: Breaking changes
- **Minor**: Backward-compatible features
- **Patch**: Bug fixes

## Deprecation Policy
- Announce deprecations with timelines.
- Provide migration guidance.
- Remove only after the deprecation period ends.
